## 2. 데이터 포맷 4가지 유형

### 포맷 A — 품목 집계형 (거제거붕백병원)
- **시트**: `Sheet` (단일)
- **헤더**: Row 1 (header=0)
- **구조**: 월 단위 품목별 판매량·금액 집계. 카테고리 소계행 + 합계행 포함
- **주요 컬럼**: 품목코드, 품목명, 품목분류, 단위, 단가, 수량, 금액, 메모
- **카테고리 코드**: `10-`식사류 / `20-`육류 / `30-`전류 / `40-`일품요리류 / `50-`반찬류 / `60-`양념류 / `70-`떡류 / `80-`제사상
- **주의사항**: 음수 값 존재(세트메뉴 할인), 소수점 수량 존재(싯가 품목)
- **파싱 코드**:
  ```python
  df = pd.read_excel(path, header=0)
  df = df[df['품목코드'].notna()]  # 소계/합계행 제거
  ```
- **분석 가능 항목**: 메뉴별 분석 O / 빈장 건수 산출 X / 객단가 산출 X / 시간대 분석 X

### 포맷 B — 빈장별 카테고리 합산형 (군산)
- **시트**: `Sheet` (단일)
- **헤더**: Row 1 (header=0)
- **구조**: 빈장(장례) 1건 = 1행. 카테고리별 합계 금액만 기록
- **주요 컬럼**: 빈소, 성명, 식사류, 반찬류, 안주류, 제사상, 후식류, 합계
- **집계행 패턴**: `[ 빈 소 ]` = 소계 / `[ 반 품 ]` = 반품 / `[ 합 계 ]` = 최종합계
- **파싱 코드**:
  ```python
  df = pd.read_excel(path, header=0)
  df = df[~df['빈소'].str.contains('소 계|반 품|합 계', na=False)]
  ```
- **분석 가능 항목**: 빈장 건수 O / 객단가 O / 메뉴별 분석 X (카테고리만) / 시간대 분석 X

### 포맷 C — 트랜잭션 상세형 표준 (서울·계명대·교원예움·대구가톨릭·삼육서울)
- **시트**: `Worksheet` (단일)
- **헤더**: Row 5 (header=4, skiprows=4). Row 1~4는 타이틀/빈행
- **구조**: 개별 상품 판매 건 = 1행. 고인번호로 빈장 그룹핑. 가장 상세한 데이터
- **데이터 규모**: 파일당 수천~4만 행
- **주요 컬럼**: 고인번호, 고인명, 빈소, 입실일자, 발인일자, 카테고리, 대분류, 중분류, 상품, 단가, 수량, 합계, 판매일자, 종교
- **서울 전용 추가 컬럼**: 나이, 성별, 단위
- **카테고리 값**: `음식`, `장례비`, `매점`
- **중분류 값 예시**: `국류`, `반찬류`, `밥류`, `과일`, `떡`, `마른안주`, `잡화 일회용품`, `상주용품`
- **파싱 코드**:
  ```python
  df = pd.read_excel(path, header=4)
  df = df.iloc[:, 1:]  # 첫 번째 빈 컬럼 제거
  df = df.dropna(subset=['고인번호'])  # 빈 행 제거
  ```
- **분석 가능 항목**: 빈장 건수 O / 메뉴별 분석 O / 객단가 O / 시간대 분석 O (판매일자 시:분 포함)

### 포맷 D — 트랜잭션 상세형 변형

#### D-안산 (안산 지점) — 조문객·결제 정보 포함
- **헤더**: Row 1 (header=0, 포맷 C와 상이)
- **고유 컬럼**: `No`(주문번호/빈장 그룹 키), `조문객`(조문객 수), `화환`(화환 수), `결제금액`, `카드합계`, `현금합계`, `코멘트`
- **분류 체계**: `분류`(=카테고리), `음식`(=상품명) — 중분류/대분류 구분 없음
- **주의**: `단가`/`수량` 컬럼 의미가 뒤바뀐 것으로 보임 (확인 필요)
- **고유 분석 가치**:
  - `조문객` 수 → 조문객 1인당 소비액 산출 가능 (전 지점 중 유일)
  - `코멘트` → 고객 피드백·선호도 텍스트 분석 가능 (전 지점 중 유일)

#### D-화성 (화성함백산) — 상조회사 정보 포함
- **헤더**: Row 5 (header=4, 포맷 C와 동일)
- **고유 컬럼**: `상조회사` (종교 컬럼 대신)
- **나머지 구조**: 포맷 C와 동일
- **고유 분석 가치**: 상조회사별 소비 패턴 분석 가능

## 4. 데이터 커버리지 및 처리 규칙

### 커버리지
- **전체 기간**: 2023년 ~ 2026년 3월
- **분석 기준 기간**: 최근 2년 (2024년 3월 ~ 2026년 3월) 권장
- **파일 형식**: Excel (.xlsx), 지점별 단일 시트

### 데이터 처리 규칙
1. **Excel 파싱**: `pandas` 우선, fallback `openpyxl`
2. **인코딩**: 한글 파일명·데이터 처리 시 UTF-8 주의
3. **결측값**: NaN·빈 셀은 분석 제외, 결측 비율 보고
4. **기간 정규화**: 지점 간 비교 시 동일 기간 기준 적용
5. **통화 단위**: 원(KRW) 기준, 천원·만원 변환 시 명시
6. **포맷 확인 필수**: 파일 파싱 전 반드시 해당 지점 포맷 유형(A/B/C/D) 확인
