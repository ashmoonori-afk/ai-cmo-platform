# 가격 전략 플레이북

## 목적

경쟁사 가격 비교와 가치 기반 분석을 결합하여 최적의 가격 포지셔닝을 도출한다. 단순한 경쟁사 가격 복사가 아닌, 고객이 인식하는 가치(Perceived Value)를 기준으로 가격을 설정하고, 티어링·번들링·프로모션 전략까지 연결하는 종합 가격 플레이북이다.

## 에이전트 조합

```
[1단계] competitor + researcher  →  병렬 실행
         competitor: 경쟁사 가격 구조·플랜·프로모션 수집
         researcher: 업종 가격 벤치마크·고객 WTP(지불의향) 데이터 수집
[2단계] strategist               →  가치 기반 가격 분석 + 티어링 설계
[3단계] reviewer                 →  가격 로직·리스크 검증
```

**실행 명령 예시:**
```
"[클라이언트명] 가격 전략 분석.
competitor: 경쟁사 {A, B, C}의 가격표, 플랜 구조, 할인 정책 수집.
researcher: {업종} 고객의 평균 WTP, 가격 민감도 데이터 수집.
strategist: 수집 데이터로 가격 포지셔닝 맵 + 추천 가격 전략 작성."
```

## 입력

| 항목 | 설명 | 필수 여부 |
|------|------|----------|
| 클라이언트 config | `clients/{client}/config.md` — 상품 목록, 현재 가격, ICP | 필수 |
| 가격 규칙 | `clients/{client}/pricing-rules.md` — 현재 가격 정책, 할인 기준 | 필수 |
| 경쟁사 목록 | `clients/{client}/config.md`의 경쟁사 + 각사 URL | 필수 |
| 고객 VOC | `knowledge-base/{client}/insights.md` — 가격에 대한 고객 피드백 | 권장 |
| 원가 구조 | 제품/서비스별 원가 (제공 가능한 경우) | 권장 |

## 참조 문서

- `clients/{client}/config.md` — 클라이언트 기본 설정
- `clients/{client}/pricing-rules.md` — 현재 가격 정책
- `playbooks/01-strategy/positioning-map.md` — 포지셔닝 기반 가격 방향성
- `prompts/shared/boilerplate.md` — 컨텍스트 로딩 규칙

## 프레임워크

### Step 1. 경쟁사 가격 비교표

competitor 에이전트가 수집한 데이터를 구조화한다:

**경쟁사 가격 비교 매트릭스:**

| 항목 | 자사 | 경쟁사 A ({이름}) | 경쟁사 B ({이름}) | 경쟁사 C ({이름}) | 업종 평균 |
|------|------|----------------|----------------|----------------|--------|
| **기본 가격 (최저 플랜)** | ₩{X} | ₩{X} | ₩{X} | ₩{X} | ₩{X} |
| **중간 플랜** | ₩{X} | ₩{X} | ₩{X} | ₩{X} | ₩{X} |
| **최상위 플랜** | ₩{X} | ₩{X} | ₩{X} | ₩{X} | ₩{X} |
| **무료 플랜/체험** | {O/X} | {O/X} | {O/X} | {O/X} | |
| **연간 결제 할인** | {X}% | {X}% | {X}% | {X}% | |
| **기업 계약 옵션** | {O/X} | {O/X} | {O/X} | {O/X} | |
| **핵심 포함 기능** | {기능 목록} | | | | |

**가격 포지셔닝 분석:**
- 자사는 현재 {프리미엄 / 중가 / 저가} 포지션
- 경쟁사 대비 가격 차이: {+X% / -X%}
- 가격 차이의 근거가 되는 기능/가치 차이: {}

---

### Step 2. 가치 기반 분석 (Value-Based Pricing)

고객이 자사 제품/서비스로 얻는 가치를 정량화한다.

**고객 가치 계산 (ROI 기반):**

| 가치 항목 | 계산 방법 | 추정값 |
|---------|---------|------|
| 시간 절약 | (절약 시간/월) × (시간당 인건비) | ₩{X}/월 |
| 비용 절감 | (대체 솔루션 비용) - (자사 비용) | ₩{X}/월 |
| 매출 증가 | (자사 활용 시 예상 추가 매출) | ₩{X}/월 |
| 리스크 감소 | (오류·실수 비용 절감) | ₩{X}/월 |
| **총 가치** | | **₩{X}/월** |
| **자사 가격** | | **₩{X}/월** |
| **가치 대비 가격 비율** | | **{X}%** |

**가치 대비 가격 비율 해석:**
- 10% 이하: 가격이 너무 낮음 — 가격 인상 또는 프리미엄 포지셔닝 검토
- 10~30%: 건강한 범위 — 현재 유지 또는 소폭 조정
- 30% 이상: 가격이 너무 높거나 가치 전달 실패 — 가격 조정 또는 가치 커뮤니케이션 강화

**고객 세그먼트별 WTP (지불 의향):**

| 세그먼트 | WTP 범위 | 현재 가격 적합성 | 접근 방법 |
|--------|---------|--------------|--------|
| 세그먼트 A (SMB) | ₩{X}~₩{X} | {적합/높음/낮음} | |
| 세그먼트 B (Mid-market) | ₩{X}~₩{X} | | |
| 세그먼트 C (Enterprise) | ₩{X}~₩{X} | | |

---

### Step 3. 가격 포지셔닝 맵

경쟁사와 자사를 가격 × 가치 2x2 맵에 배치한다:

```
가치 높음
^
|
|   [고가치·저가]           [고가치·고가]
|   (언더프라이싱)          (프리미엄 포지션)
|
|   {경쟁사 B}             ★ {자사명} (목표)
|                           {경쟁사 A}
|
+---------------------------------> 가격 높음
|
|   [저가치·저가]           [저가치·고가]
|   (저가 경쟁)             (오버프라이싱 위험)
|
|   {경쟁사 C}
|
가치 낮음
```

**현재 포지션 평가:**
- 현재 자사 위치: {사분면}
- 목표 포지션: {사분면}
- 이동 전략: {가격 조정 / 가치 강화 / 커뮤니케이션 개선}

---

### Step 4. 가격 구조 설계 (티어링 & 번들링)

**추천 플랜 구조:**

#### 플랜 1: {스타터/베이직} — ₩{X}/월

| 포함 항목 | 제한 |
|---------|------|
| {기능 A} | {횟수/용량 제한} |
| {기능 B} | |

**타겟:** {세그먼트 A} — 가격 민감도 높고 핵심 기능만 필요한 초기 사용자
**전환 목표:** 3개월 내 상위 플랜으로 업그레이드

---

#### 플랜 2: {스탠다드/프로} — ₩{X}/월 ⭐ 추천

| 포함 항목 | 제한 |
|---------|------|
| {기능 A} | 무제한 |
| {기능 B} | |
| {기능 C} (추가) | |

**타겟:** {세그먼트 B} — 성장 중인 팀, 핵심 KPI 개선 필요
**앵커링 전략:** 최상위 플랜과의 차이를 명확히 하여 이 플랜의 가성비를 부각

---

#### 플랜 3: {엔터프라이즈/프리미엄} — ₩{X}/월 또는 문의

| 포함 항목 | 제한 |
|---------|------|
| 플랜 2 전체 | |
| {고급 기능 A} | |
| {고급 기능 B} | |
| 전담 지원 | |

**타겟:** {세그먼트 C} — 대기업·고가치 계정
**가격 협상 여지:** 연간 계약 시 {X}% 추가 할인

---

**번들링 전략:**

| 번들명 | 포함 상품/서비스 | 개별 합산가 | 번들가 | 할인율 |
|------|--------------|----------|------|------|
| 번들 A | {상품 1 + 상품 2} | ₩{X} | ₩{X} | {X}% |
| 번들 B | | | | |

**번들링 설계 원칙:**
- 번들 할인율: 15~25% (고객 가치 인식 vs 마진 균형)
- 번들 내 핵심 상품은 단독 구매보다 번들이 합리적으로 느껴지도록 설계
- 번들 구매 시 추가 혜택 (예: 우선 지원, 맞춤 온보딩) 포함하여 가치 극대화

---

### Step 5. 가격 앵커링 & 프레이밍 전략

**앵커링 기법:**
- 최상위 플랜을 먼저 제시하여 중간 플랜이 합리적으로 보이게 설정
- "시간당 ₩{X}" 또는 "직원 1인당 ₩{X}" 등 단위 환산으로 저렴함 인지
- 연간 결제 시 절약 금액을 월 환산으로 표시: "연간 결제 시 ₩{X}/월 절약"

**프레이밍 기법:**
- 손실 회피: "X를 놓치지 마세요" vs "X를 얻으세요" — 업종·타겟에 맞게 선택
- 사회적 증거: "이 플랜을 선택한 {X}%의 고객" 표현 활용
- 가치 비교: "커피 한 잔 값으로 {핵심 혜택}"

---

### Step 6. 프로모션 전략 & 캘린더

**프로모션 유형별 가이드라인:**

| 프로모션 유형 | 최대 할인율 | 사용 조건 | 주의 사항 |
|------------|----------|---------|---------|
| 신규 가입 할인 | {X}% | 첫 구매 / 첫 달 | 반복 사용 방지 (계정당 1회) |
| 연간 결제 할인 | {X}% | 연간 일괄 결제 시 | 환불 정책 명확히 |
| 소개 리워드 | ₩{X} 크레딧 | 지인 추천 + 가입 시 | 양방향 (추천인+피추천인) |
| 시즌 프로모션 | {X}% | 특정 시즌/이벤트 | 연 {X}회 이하로 제한 |
| 번들 패키지 | {X}% | 특정 상품 조합 | 마진율 {X}% 이상 유지 |

**프로모션 캘린더:**

| 월 | 프로모션 | 할인율/혜택 | 타겟 | 예상 효과 |
|---|---------|----------|------|--------|
| 1월 | 신년 혜택 | | | |
| 3월 | 봄 시즌 | | | |
| 6월 | 상반기 마감 | | | |
| 9월 | 가을 시즌 | | | |
| 11월 | 블랙프라이데이 | | | |
| 12월 | 연말 혜택 | | | |

**프로모션 실행 원칙:**
- 마진율 {X}% 이하로 내려가는 프로모션 금지
- 프로모션 기간: 2주 이내 (상시 할인은 가격 앵커 하락 위험)
- 프로모션 종료 후 정가 구매 유도 이메일 시퀀스 필수

---

### Step 7. 가격 인상/조정 시나리오

**가격 인상 트리거 (이 조건 충족 시 가격 인상 검토):**
- 신규 고객 유입이 목표 대비 120% 이상 (수요 초과)
- 경쟁사 가격이 자사보다 평균 20% 이상 높아진 경우
- 제품 주요 기능 업그레이드 후

**가격 인상 시 커뮤니케이션 플로우:**
1. 기존 고객에게 30일 전 사전 공지 이메일
2. 가격 인상 이유 (기능 추가·비용 증가) 명확히 설명
3. 기존 고객에게 연간 결제 전환 시 현재 가격 적용 옵션 제공
4. 인상 7일 전 리마인드 이메일 발송
