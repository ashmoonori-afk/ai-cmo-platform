# GTM 모션 분석 플레이북

## 목적

7가지 GTM 모션(Inbound, Outbound, Paid, Community, Referral, Partnership, PLG/제휴)을 클라이언트 비즈니스 특성에 맞게 스코어링하고, 실행 가능한 모션 스택과 90일 로드맵을 도출한다. 채널 의존도를 분산시키고, 성장 단계·팀 규모·예산에 맞는 최우선 모션 3개를 선택하는 것이 핵심 목표다.

## 에이전트 조합

```
[1단계] researcher + competitor  →  병렬 실행 (시장·경쟁 데이터 수집)
[2단계] strategist               →  스코어링 + 모션 스택 결정
[3단계] reviewer                 →  논리 검증 + 실행 가능성 확인
```

**실행 명령 예시:**
```
"[클라이언트명] GTM 모션 분석 실행.
researcher: 해당 업종 주요 채널 효과 데이터 수집.
competitor: 경쟁사 채널 전략 분석.
결과를 strategist에게 전달하여 7개 모션 스코어링 후 상위 3개 선정."
```

## 입력

| 항목 | 설명 | 필수 여부 |
|------|------|----------|
| 클라이언트 config | `clients/{client}/config.md` — 업종, 상품 단가, 구매 주기, ICP | 필수 |
| 현재 채널 데이터 | 현재 사용 중인 채널, 트래픽 출처, 채널별 전환율 | 필수 |
| 팀 규모 & 예산 | 마케팅 팀 인원 수, 월간 마케팅 예산 범위 | 필수 |
| 경쟁사 정보 | `clients/{client}/config.md`의 경쟁사 목록 | 권장 |
| 이전 실험 결과 | `knowledge-base/{client}/lessons-learned.md` | 권장 |

## 참조 문서

- `clients/{client}/config.md` — 클라이언트 기본 설정
- `prompts/shared/boilerplate.md` — 컨텍스트 로딩 규칙

## 프레임워크

### Step 1. 비즈니스 특성 프로파일링

분석 시작 전 다음 항목을 명확히 확인한다:

| 항목 | 클라이언트 값 | GTM 함의 |
|------|-------------|---------|
| 상품 단가 | ₩{X} | 단가 낮으면 PLG/Inbound 유리, 높으면 Outbound/ABM 유리 |
| 판매 주기 | {X}일 | 단기면 Paid/Inbound, 장기면 Outbound/ABM |
| B2B/B2C 비중 | B2B {X}% / B2C {X}% | B2B 비중 높을수록 Outbound/Partnership 가중치 ↑ |
| 재구매 주기 | 월 {X}회 / 연 {X}회 | 재구매 짧으면 Retention/PLG, 길면 Referral |
| 팀 규모 | {X}명 | 소팀은 채널 집중, 대팀은 다채널 |

---

### Step 2. 7가지 GTM 모션 스코어카드

각 모션을 4개 기준(현재 활용도, 성장 잠재력, 투자 대비 효과, 실행 난이도)으로 평가한다.
**점수 기준:** 1-10점 (10점 = 해당 클라이언트에 최적)

| GTM 모션 | 종합 점수 (1-10) | 현재 활용도 | 성장 잠재력 | 투자 대비 효과(ROI) | 실행 난이도 | 우선순위 |
|---------|--------------|-----------|-----------|-----------------|-----------|--------|
| 1. 인바운드 (SEO·블로그·콘텐츠) | | /10 | /10 | /10 | /10 (낮을수록 쉬움) | |
| 2. 아웃바운드 (콜드메일·영업 전화) | | /10 | /10 | /10 | /10 | |
| 3. 유료 광고 (검색·디스플레이·SNS) | | /10 | /10 | /10 | /10 | |
| 4. 커뮤니티 (이벤트·웨비나·온라인 커뮤니티) | | /10 | /10 | /10 | /10 | |
| 5. 리퍼럴 (고객 추천·입소문) | | /10 | /10 | /10 | /10 | |
| 6. 파트너십 (공동 마케팅·채널 파트너) | | /10 | /10 | /10 | /10 | |
| 7. 제휴/PLG (복지몰·플랫폼 입점·제품 주도) | | /10 | /10 | /10 | /10 | |

**종합 점수 계산식:**
```
종합 점수 = (현재 활용도 × 0.2) + (성장 잠재력 × 0.35) + (ROI × 0.35) + ((10 - 실행 난이도) × 0.1)
```

---

### Step 3. 모션별 판단 근거

각 모션에 대해 아래 템플릿으로 서술한다:

#### [모션명] — 점수: X / 10

**판단 근거 (3-5줄):**
- 해당 클라이언트의 업종·ICP·구매 주기 기준으로 왜 이 점수인가

**핵심 전술 (3-5개 bullet):**
- 구체적인 실행 방법 (플랫폼, 콘텐츠 유형, 타겟 세그먼트)

**적합 타겟 페르소나:**
- ICP 중 이 모션에 가장 잘 반응하는 세그먼트

**주요 리스크:**
- 이 모션 실행 시 주의해야 할 함정

---

### Step 4. 상위 3개 모션 선택 기준

아래 질문에 답하며 최종 모션 스택을 결정한다:

```
Q1. 지금 당장 매출이 필요한가?
    → Yes: Outbound + Paid 우선
    → No: Inbound + Partnership 장기 투자

Q2. 고객이 구매 결정 순간에 어디에 있는가?
    → 검색: Inbound + Paid
    → 특정 플랫폼/앱: Partnership + 제휴
    → 영업 접점: Outbound + ABM

Q3. 팀 실행 역량은?
    → 콘텐츠 강점: Inbound + Community
    → 영업 강점: Outbound + ABM
    → 기술 강점: PLG + 제휴

Q4. 경쟁사가 이미 장악한 채널은?
    → 경쟁사 공백 채널을 우선 공략
```

**최종 모션 스택:**
```
[1순위] {모션명} ── {이유 한 줄}
[2순위] {모션명} ── {이유 한 줄}
[3순위] {모션명} ── {이유 한 줄}
[보조]  {모션명} ── {제한적 활용 조건}
```
