# 퍼널 설계 플레이북

## 목적

고객의 구매 여정(인지 → 관심 → 고려 → 전환 → 유지 → 추천)을 단계별로 설계하고, 각 단계에서 적합한 채널·콘텐츠·전술·전환 목표를 정의한다. 퍼널 내 병목 구간(Bottleneck)을 발견하고, 각 단계 간 전환율을 개선하기 위한 구체적인 액션을 도출하는 것이 목적이다.

## 에이전트 조합

```
[1단계] strategist   →  퍼널 단계별 설계 + 전환율 목표 설정
[2단계] reviewer     →  퍼널 로직 검증 + 병목 구간 식별
```

**실행 명령 예시:**
```
"[클라이언트명] 퍼널 설계.
현재 전환율 데이터: 방문자 {X}명 → 가입 {X}명 → 구매 {X}명.
strategist: 6단계 퍼널 설계 + 단계별 목표 전환율 + 콘텐츠 매핑.
reviewer: 현재 데이터 기준 병목 구간 분석 + 개선 우선순위 제안."
```

## 입력

| 항목 | 설명 | 필수 여부 |
|------|------|----------|
| 클라이언트 config | `clients/{client}/config.md` — ICP, 구매 주기, 핵심 채널 | 필수 |
| 현재 퍼널 데이터 | 단계별 방문자·전환·이탈 수치 (있는 경우) | 권장 |
| 채널 전략 | `channel-strategy` 출력물 또는 현재 채널 목록 | 권장 |
| 포지셔닝 메시지 | `positioning-map` 출력물의 메시지 프레임워크 | 권장 |
| 고객 VOC | `knowledge-base/{client}/insights.md` — 이탈 이유, 구매 동인 | 권장 |

### Step 2. 단계별 퍼널 상세 설계

#### Stage 1. 인지 (Awareness)

**목표:** 타겟 고객이 자사 브랜드·제품을 처음 인식하게 만든다

| 항목 | 내용 |
|------|------|
| 고객 상태 | 문제는 인식하지만 해결책을 모른다 / 또는 문제 자체를 모른다 |
| 핵심 질문 | "이 문제를 어떻게 해결하지?" / "좋은 방법이 있을까?" |
| 자사 역할 | 문제 정의 + 해결 가능성 제시 |

**채널 & 전술:**
| 채널 | 콘텐츠 유형 | 빈도 | 목표 지표 |
|------|-----------|------|--------|
| SEO / 네이버 블로그 | 문제 기반 롱폼 콘텐츠 | 주 2-3회 | 오가닉 방문자 |
| 유료 검색 광고 | 문제 키워드 텍스트 광고 | 상시 | 클릭 수, CTR |
| SNS 광고 | 브랜드 인지 이미지/영상 | 캠페인 기간 | 노출, 도달 |
| PR / 언론 | 업종 전문 기사·인터뷰 | 월 1-2회 | 기사 수, 언급 |

**핵심 메시지 방향:**
- 솔루션이 아닌 문제를 중심으로 커뮤니케이션
- "혹시 이런 문제 겪고 계신가요?" 형태의 공감 유도
- 브랜드 노출보다 콘텐츠 가치 우선

**전환 목표 (인지 → 관심):**
- 방문자 중 {X}%가 2페이지 이상 방문
- 블로그 방문자 중 {X}%가 구독 또는 SNS 팔로우

#### Stage 2. 관심 (Interest)

**목표:** 브랜드를 기억하고 지속적으로 콘텐츠를 소비하게 만든다

| 항목 | 내용 |
|------|------|
| 고객 상태 | 브랜드를 알고 있으나 아직 구체적 행동을 하지 않음 |
| 핵심 질문 | "이 회사가 믿을 만한가?" / "이게 내 상황에 맞는가?" |
| 자사 역할 | 전문성 증명 + 신뢰 구축 |

**채널 & 전술:**
| 채널 | 콘텐츠 유형 | 빈도 | 목표 지표 |
|------|-----------|------|--------|
| 이메일 뉴스레터 | 인사이트 큐레이션 | 격주 | 오픈율, 구독 유지율 |
| LinkedIn | 전문성 포스트, 데이터 인사이트 | 주 3-5회 | 팔로워 증가, 공유 |
| 유튜브 / 웨비나 | 교육 콘텐츠, 사례 공유 | 월 1-2회 | 조회수, 등록률 |
| 리타겟팅 광고 | "더 알아보기" 콘텐츠 광고 | 상시 | CTR, 재방문율 |

**리드 마그넷 설계:**
- 유형: {체크리스트 / 가이드 / 템플릿 / 워크시트} — `03-content/lead-magnet.md` 참조
- 주제: 타겟 고객의 핵심 페인포인트 해결 → {구체적 주제}
- 게이트 조건: 이메일 주소 수집 (가능한 경우 직함·회사 추가)

**전환 목표 (관심 → 고려):**
- 이메일 구독자 중 {X}%가 리드 마그넷 다운로드
- 웨비나 등록자 중 {X}%가 이후 데모 요청

#### Stage 3. 고려 (Consideration)

**목표:** 자사를 경쟁사와 비교·평가하는 단계에서 최우선 선택지가 된다

| 항목 | 내용 |
|------|------|
| 고객 상태 | 솔루션을 검색하고 옵션을 비교 중 |
| 핵심 질문 | "이게 다른 것보다 나은가?" / "ROI가 있는가?" / "리스크가 없는가?" |
| 자사 역할 | 차별점 증명 + 구매 장벽 제거 |

**채널 & 전술:**
| 채널 | 콘텐츠 유형 | 빈도 | 목표 지표 |
|------|-----------|------|--------|
| 이메일 너처링 시퀀스 | 케이스스터디, ROI 계산기 | 주 1-2회 | 링크 클릭율 |
| 웹사이트 (비교 페이지) | vs 경쟁사 비교표 | 상시 업데이트 | 페이지 방문, 체류시간 |
| 세일즈 콜/데모 | 맞춤 데모, Q&A | 수요 기반 | 데모 → 견적 전환율 |
| 고객 리뷰/케이스스터디 | 실제 사용자 후기, 수치 | 지속 수집 | 리뷰 수, 평점 |

**고려 단계 핵심 자산 목록:**
- [ ] 케이스스터디 최소 3개 (업종별 1개씩) — `03-content/case-study.md` 참조
- [ ] 경쟁사 비교표 (웹사이트 내 별도 페이지)
- [ ] ROI 계산기 또는 절약 추정 도구
- [ ] 자주 묻는 질문 (FAQ) 페이지
- [ ] 30일 무료 체험 또는 파일럿 제안 (해당 시)

**전환 목표 (고려 → 전환):**
- 데모 신청자 중 {X}%가 견적 요청
- 무료 체험 가입자 중 {X}%가 유료 전환

#### Stage 4. 전환 (Conversion)

**목표:** 구매·가입·계약 완료. 마찰(Friction)을 최소화하고 의사결정을 가속화한다

| 항목 | 내용 |
|------|------|
| 고객 상태 | 구매 의향 있으나 마지막 장벽 존재 (가격, 조건, 내부 승인 등) |
| 핵심 질문 | "지금 당장 해야 하나?" / "이 조건이 최선인가?" |
| 자사 역할 | 마지막 장벽 제거 + 행동 유도 |

**채널 & 전술:**
| 채널 | 콘텐츠 유형 | 빈도 | 목표 지표 |
|------|-----------|------|--------|
| 랜딩 페이지 | 명확한 CTA + 사회적 증거 | A/B 테스트 상시 | 랜딩 → 전환율 |
| 이메일 (전환 시퀀스) | 긴급성 + 마지막 제안 | 3통 시퀀스 | 이메일 → 구매율 |
| 세일즈 팔로업 | 개인화 제안서 | 즉시 대응 | 제안 → 계약 전환율 |
| 리타겟팅 (전환 최적화) | 특가 / 한정 혜택 | 집중 기간 | ROAS, CVR |

**전환 장벽 제거 체크리스트:**
- [ ] 결제 단계 최소화 (3단계 이하)
- [ ] 신뢰 요소 (SSL, 리뷰, 환불 정책) 전환 페이지에 노출
- [ ] 가격 저항 해소: 월 환산 표시, 타 비용 대비 비교
- [ ] 긴급성 요소: 한정 혜택, 마감 기한 (남용 금지)
- [ ] 이탈 방지: Exit Intent 팝업 (10% 할인 쿠폰 등)

**전환 목표:**
- 랜딩 페이지 전환율: {X}% (업종 평균 대비)
- 이메일 시퀀스 → 구매율: {X}%
- 전체 퍼널 전환율 (최초 방문 → 첫 구매): {X}%

#### Stage 5. 유지 (Retention)

**목표:** 첫 구매 후 재구매·갱신·업셀을 유도한다. 기존 고객 LTV 극대화

| 항목 | 내용 |
|------|------|
| 고객 상태 | 첫 구매 후 가치를 경험 중 (또는 실망 중) |
| 핵심 질문 | "계속 쓸 만한가?" / "더 많이 투자할 가치가 있는가?" |
| 자사 역할 | 온보딩 성공 + 지속 가치 전달 + 업셀 타이밍 포착 |

**채널 & 전술:**
| 채널 | 콘텐츠 유형 | 빈도 | 목표 지표 |
|------|-----------|------|--------|
| 온보딩 이메일 시퀀스 | 사용법 가이드, 팁 | 구매 후 4주 | 온보딩 완료율 |
| 인앱 / 제품 내 알림 | 기능 안내, 진행 상황 | 사용 패턴 기반 | 기능 활성화율 |
| 고객 성공 콜 | QBR(분기 비즈니스 리뷰) | 분기 1회 | 재계약율, NPS |
| 이메일 (재구매 유도) | 리마인더 + 신기능 안내 | 월 1-2회 | 재구매율 |
| 커뮤니티 / VIP 프로그램 | 독점 콘텐츠, 혜택 | 지속 | 참여율, LTV |

**온보딩 시퀀스 구조 (4주):**
- Day 0: 환영 이메일 + 첫 번째 핵심 기능 안내
- Day 3: 두 번째 기능 + 성공 사례 공유
- Day 7: 진행 상황 확인 + 추가 도움 제안
- Day 14: 중간 점검 + 업그레이드 혜택 소개
- Day 30: 첫 달 성과 요약 + 재구매/연장 CTA

- 30일 유지율: {X}%
- 90일 유지율: {X}%
- 연간 갱신율 (B2B): {X}%
- 업셀 전환율: {X}%
