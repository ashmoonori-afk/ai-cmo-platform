# 고객 피드백 분석 플레이북

## 목적

월간 단위로 다양한 채널에서 수집된 고객 피드백을 테마별로 분류하고, 감성 분석과 빈도×심각도 우선순위 매트릭스를 통해 제품·서비스 개선 및 마케팅 메시지 조정에 활용 가능한 인텔리전스를 생성한다.

빈도: 월 1회 (또는 주요 피드백 이벤트 발생 시 수동 실행)

## 에이전트 조합

| 역할 | 에이전트 | 역할 설명 |
|------|---------|---------|
| 데이터 분석 | `data-analyst` | 피드백 수집·분류·감성 분석·우선순위 매트릭스 작성 |

실행 순서: `data-analyst` (단일, 5단계 순차 실행)

## 입력

| 항목 | 필수 여부 | 설명 |
|------|---------|------|
| `feedback_raw` | 필수 | 원시 피드백 데이터 (CSV, JSON, 또는 텍스트 목록) |
| `feedback_sources` | 필수 | 피드백 수집 채널 목록 (아래 수집원 정의 참조) |
| `analysis_period` | 선택 | 기본값: 최근 30일 |
| `previous_analysis` | 선택 | 직전 월 분석 결과 — 트렌드 비교용 |

입력이 없을 경우: "피드백 데이터가 없습니다. 수집 채널을 확인하거나 데이터를 제공해 주세요." 안내 후 중단.

### 민감정보 입력 Gate

- 분석 전 `feedback_raw`에서 고객명, 전화번호, 이메일, 주민/사업자번호, 계정 ID, 쿠키, 토큰, API 키, 인증 헤더를 제거하거나 마스킹한다.
- 이메일, 채팅, 헬프데스크, 설문, 미팅 메모는 원문 전체를 출력하지 않고 익명화된 짧은 인용과 테마/빈도/심각도만 남긴다.
- private CRM row, GA 사용자 단위 데이터, 원본 CSV/Excel 행은 산출물에 붙이지 않는다.
- 민감정보가 제거되지 않은 입력이면 중단하고 redacted working copy를 요청한다.

## 프레임워크

### Step 1: 피드백 수집원 정의

분석 전 피드백 소스를 확인하고 목록화한다.

| 소스 유형 | 채널 예시 | 피드백 형태 |
|---------|---------|---------|
| 고객 서포트 | 이메일, 채팅, 헬프데스크 티켓 | 문의·불만·요청 텍스트 |
| 리뷰 플랫폼 | G2, Capterra, Trustpilot, 앱스토어 | 별점 + 텍스트 리뷰 |
| NPS/CSat 설문 | 분기별 설문, 온보딩 후 설문 | 점수 + 서술형 답변 |
| 영업 인터뷰 | Win/Loss 인터뷰, 갱신 미팅 메모 | SAL-03 출력 활용 |
| SNS·커뮤니티 | LinkedIn, Twitter, 슬랙 커뮤니티 | 공개 게시물·댓글 |
| 내부 CS 메모 | 고객 성공 매니저 주간 기록 | 정성적 관찰 |

소스별 피드백 건수와 기간을 기록한다.

### Step 2: 테마 분류

수집된 피드백을 아래 4개 주요 테마로 1차 분류하고, 하위 테마로 세분화한다.

**주요 테마:**

| 테마 | 하위 테마 예시 |
|------|-------------|
| 기능 (Feature) | 특정 기능 요청, 버그·오류, UX 불편, 성능 |
| 가격 (Pricing) | 가격 대비 가치, 요금제 구조, 계약 조건 |
| 지원 (Support) | 응답 속도, 전문성, 온보딩 경험, 문서 품질 |
| 경쟁 (Competitive) | 경쟁사 비교 언급, 전환 이유, 재계약 거절 이유 |

분류 불가 피드백은 `기타` 테마로 분류 후 별도 검토.

각 피드백에 다음 메타데이터를 태그한다:
- `theme`: 주요 테마
- `sub_theme`: 하위 테마
- `source`: 피드백 소스
- `customer_segment`: ICP 적합도 (ICP 고객 / 비ICP 고객 / 미확인)

### Step 3: 빈도 분석

테마·하위 테마별 피드백 건수를 집계한다.

| 테마 | 하위 테마 | 건수 | 전월 대비 | 전체 비중 |
|------|---------|------|---------|---------|
| 기능 | UX 불편 | {n} | {+/-n%} | {%} |
| 기능 | 기능 요청 | {n} | {+/-n%} | {%} |
| 가격 | 가격 대비 가치 | {n} | {+/-n%} | {%} |
| 지원 | 응답 속도 | {n} | {+/-n%} | {%} |
| 경쟁 | 경쟁사 언급 | {n} | {+/-n%} | {%} |

### Step 4: 감성 분석

각 피드백을 긍정/중립/부정으로 분류하고, 테마별 감성 분포를 집계한다.

감성 분류 기준:
- **긍정**: 만족·칭찬·추천 의사 표현
- **중립**: 정보 요청, 사실 확인, 기능 문의
- **부정**: 불만·실망·해지 위협·경쟁사 전환 의사

| 테마 | 긍정 | 중립 | 부정 | 감성 점수 (–100~+100) |
|------|------|------|------|---------------------|
| 기능 | {%} | {%} | {%} | {점수} |
| 가격 | {%} | {%} | {%} | {점수} |
| 지원 | {%} | {%} | {%} | {점수} |
| 경쟁 | {%} | {%} | {%} | {점수} |

감성 점수 = (긍정 건수 - 부정 건수) / 전체 건수 × 100

### Step 5: 우선순위 매트릭스 (빈도 × 심각도)

각 하위 테마를 빈도와 심각도 두 축으로 평가하여 4분면 매트릭스에 배치한다.

**심각도 기준:**
- **높음**: 계약 해지 또는 갱신 거절과 직결, 다수 ICP 고객에게 반복 언급
- **중간**: 고객 만족도에 영향을 주나 즉각적 해지로 이어지지 않음
- **낮음**: 소수 피드백, 비ICP 고객 중심, 사업 영향 미미

**우선순위 매트릭스:**

```
          낮은 빈도          높은 빈도
높은 심각도  [ 즉시 조사 ]     [ 즉시 대응 ] ← 최우선
낮은 심각도  [ 장기 검토 ]     [ 개선 큐 ]
```

| 우선순위 | 하위 테마 | 빈도 | 심각도 | 권장 대응 |
|---------|---------|------|------|---------|
| 즉시 대응 | {테마} | 높음 | 높음 | {구체적 대응 방안} |
| 즉시 조사 | {테마} | 낮음 | 높음 | {원인 분석 후 판단} |
| 개선 큐 | {테마} | 높음 | 낮음 | {로드맵 반영 검토} |
| 장기 검토 | {테마} | 낮음 | 낮음 | {분기 리뷰 시 재검토} |
