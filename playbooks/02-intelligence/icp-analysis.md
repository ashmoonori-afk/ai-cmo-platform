# ICP 분석 플레이북

## 목적

분기 1회 실제 고객 데이터를 분석하여 ICP(Ideal Customer Profile)를 정의하거나 업데이트한다. 현재 ICP 정의와 실제 성공 고객 데이터 간의 갭을 파악하고, 영업·마케팅 타겟팅의 정밀도를 높이는 것이 목표다.

빈도: 분기 1회 (또는 고객 데이터가 유의미하게 변경되었을 때 수동 실행)

## 에이전트 조합

| 역할 | 에이전트 | 역할 설명 |
|------|---------|---------|
| 데이터 분석 | `data-analyst` | 고객 데이터 패턴 분석 (인구통계, 행동, 구매 이력) |
| 시장 조사 | `researcher` | 외부 시장 데이터·트렌드 수집 (병렬 실행) |
| 전략 종합 | `strategist` | data-analyst + researcher 결과를 종합하여 ICP 업데이트 제안 |

실행 순서: `data-analyst` + `researcher` (병렬) → `strategist` (순차)

## 입력

| 항목 | 필수 여부 | 설명 |
|------|---------|------|
| `current_icp` | 필수 | 현재 ICP 정의 (`references/icp-definition.md` 또는 CLAUDE.md config.icp) |
| `customer_data` | 필수 | CRM 데이터 — 성사 딜, 손실 딜, 고객 체류 기간, ARR/거래 규모 |
| `win_loss_notes` | 선택 | 최근 분기 Win/Loss 인터뷰·메모 |
| `market_data` | 선택 | 업계 보고서, 시장 규모 데이터 |

입력 데이터가 없을 경우: "CRM 데이터 또는 현재 ICP 정의 파일이 없습니다. 제공해 주세요." 안내 후 중단.

## 프레임워크

### Step 1: 현재 ICP 정의 검토 (strategist)

`references/icp-definition.md`에서 현재 ICP를 읽고 다음 4개 차원으로 정리한다.

| 차원 | 현재 ICP 정의 |
|------|-------------|
| 인구통계 (Firmographic) | 회사 규모, 산업, 지역, 직책 |
| 행동 (Behavioral) | 구매 전 행동, 사용 패턴, 계약 갱신율 |
| 심리 (Psychographic) | 비즈니스 가치관, 성장 지향성, 기술 수용성 |
| 구매 동기 (Motivational) | 주요 페인포인트, 구매 트리거, 의사결정 기준 |

### Step 2: 실제 고객 데이터 분석 (data-analyst)

CRM 데이터에서 다음 패턴을 분석한다.

**2a. 성공 고객 패턴 (Won Deals)**
- 공통 산업·회사 규모·직책
- 평균 계약 규모 (ACV/ARR)
- 영업 사이클 길이
- 최초 접촉 ~ 계약까지 주요 마일스톤
- 가장 많이 언급된 구매 이유 (Win 이유)

**2b. 실패 고객 패턴 (Lost Deals)**
- 주로 손실이 발생하는 산업·회사 규모·직책
- 주요 손실 이유 (가격, 기능, 경쟁사, 타이밍)
- 손실 단계 (어느 영업 단계에서 주로 이탈하는가)

**2c. 고객 유지율 패턴 (Retention)**
- 갱신율이 높은 고객 세그먼트 특성
- 이탈 고객의 공통점
- NPS 또는 CSat 점수와 고객 특성 간 상관관계

**2d. 확장 매출 패턴 (Expansion)**
- 업셀·크로스셀이 발생한 고객 특성
- 확장 계약이 일어난 타이밍과 트리거

### Step 3: 외부 시장 데이터 수집 (researcher, 병렬)

- 자사 타겟 산업의 성장률 및 주요 플레이어 변화
- 경쟁사의 타겟 고객 세그먼트 변화 (경쟁사 매트릭스 참조)
- 새롭게 부상하는 TAM/SAM 기회 세그먼트

### Step 4: 갭 분석 (strategist)

현재 ICP 정의와 실제 데이터 분석 결과를 비교한다.

| 항목 | 현재 ICP | 실제 성공 고객 패턴 | 갭 |
|------|---------|----------------|----|
| 회사 규모 | {현재 정의} | {실제 패턴} | {일치/수정 필요} |
| 산업 | {현재 정의} | {실제 패턴} | {일치/수정 필요} |
| 직책 | {현재 정의} | {실제 패턴} | {일치/수정 필요} |
| 페인포인트 | {현재 정의} | {실제 패턴} | {일치/수정 필요} |
| 구매 트리거 | {현재 정의} | {실제 패턴} | {일치/수정 필요} |

### Step 5: ICP 업데이트 제안 (strategist)

갭 분석 결과를 바탕으로 업데이트된 ICP를 제안한다.

필요시 ICP를 1차 ICP(최우선 타겟)와 2차 ICP(부차 타겟)로 계층화한다.
