# 네이버 스마트플레이스 세팅·운영 플레이북

## 목적

전 업종 공통 1순위 로컬 채널인 네이버 스마트플레이스를 **사장님 대신** 세팅하고, 한 달 치 운영물까지 만들어 납품한다. 산출물은 복사–붙여넣기 가능한 완성 상태(`prompts/shared/deliverable-standard.md` 적용).

근거: 네이버 검색 점유율 80%+ (파이낸셜뉴스 2026, fnnews.com/news/202606211826183635), 네이버 지도 MAU 약 2,650만·등록 장소 238만 (뉴시스 2025-03, newsis.com/view/NISX20250321_0003108304). 미등록 = 동네 검색에서 존재하지 않는 가게.

## 에이전트 조합

```
seo-specialist → copywriter → reviewer
```

| 순서 | 에이전트 | 역할 | 모델 |
|------|---------|------|------|
| 1 | seo-specialist | 대표키워드 5개 선정 + 노출 요인 진단 | sonnet |
| 2 | copywriter | 설명문·메뉴·소식·답글 등 전 문안 작성 | sonnet |
| 3 | reviewer | 실행성 검증 (deliverable-standard 3.5단계) + 어뷰징 제로 확인 | sonnet |

## 입력

| 항목 | 설명 | 필수 |
|------|------|------|
| `client` | 클라이언트 폴더명 | 필수 |
| `place_status` | 신규 등록 / 이미 등록(URL) | 필수 |
| `menu_list` | 메뉴(상품)명 + 가격 목록 | 필수 |

## 프레임워크

### Step 1: 등록 상태 진단 (신규면 등록 대행 시트)

신규 등록이면 사장님이 화면 단위로 따라 할 수 있는 등록 시트를 만든다:
smartplace.naver.com 로그인 → 신규 등록 → 사업자등록증 정보 → 기본정보 → 심사(2시간~5일).
필수 입력값(업체명/전화/주소/업종)을 **미리 채운 표**로 제공한다. 휴대폰 인증은 네이버 ID에 등록된 번호로만 가능함을 안내.
(절차 출처: 오터포스 가이드 tryotter.co.kr/articles/naver-smartplace-registration-guide, 오픈애즈 openads.co.kr/content/contentDetail?contsId=4761)

### Step 2: 대표키워드 5개 (seo-specialist)

- "지역명+업종" 대중 키워드 + 특색 메뉴/서비스명 조합으로 5개 모두 채운다 (출처: 오터포스, 위 URL)
- 대표키워드는 검색어 등록이 아니라 **연관성 신호**임을 산출물에 명시 (초보 오해 1위 — 오픈애즈, 위 URL)
- 각 키워드에 선정 근거 1줄 (검색 수요·경쟁 강도는 확인 방법과 함께)

### Step 3: 문안 전량 작성 (copywriter)

1. 업체 설명문 300자 — 업종+지역+차별점, 과장·최상급 금지
2. 메뉴/상품 설명 (메뉴당 50자) + 사진 촬영 지시문 (메뉴당 1줄: 각도/배경/시간대)
3. 찾아오는 길 + 주차 안내문
4. 예약/톡톡 인사말 + 부재중 자동응답 문구

### Step 4: 소식 4주분 + 리뷰 답글 세트

- 소식 게시물 주 1회 × 4건 (이벤트/신메뉴/휴무/가게 스토리) — "활성화된 업체" 가점 근거 (모아라이브, moalive.org/학원-네이버-스마트플레이스-마케팅-방법-10가지/)
- 영수증 리뷰 답글 템플릿 10종: 긍정 5 / 불만 3 / 무플(내용 없는 별점) 2 — 부정 리뷰는 24시간 내 답글 원칙
- 악성 리뷰 신고 판단 기준표 (신고 처리 3영업일 — liveinthehome.com 2025 가이드)
- 포스 연동 시 결제 직후 리뷰 요청 자동화 가능 안내 (토스플레이스 tossplace.com/story/naver_receipt_review)

### Step 5: 어뷰징 제로 게이트 (reviewer)

- 2025-05 알고리즘 개편: 계량 지표 → **실사용자 행동 패턴**(메뉴 클릭·예약·길찾기·전화·방문) 중심 (하랑마케팅 2025, keyzard.cc/harangmarketing/nb/223864602749)
- 트래픽 구매·가짜 저장은 유입 구조·비율 분석으로 역효과 위험 [추정 — 페널티 세부 기준 비공개]
- 산출물에 저장하기 유도 문구는 **실제 혜택 연결형**만 허용 (예: "저장하고 오시면 ○○")

## 산출물 (한 번에 납품하는 묶음)

```
1. 사장님용 한 장 요약 (이게 뭔지 / 오늘 할 일 하나)
2. 등록/세팅 시트 (입력값 미리 채움, 화면 단위 스텝)
3. 대표키워드 5개 + 근거
4. 업체 설명문 300자 + 메뉴 문안 전량 + 길/주차 안내
5. 소식 4주분 (날짜 캘린더 포함)
6. 리뷰 답글 템플릿 10종 + 신고 기준표
7. 월간 리포트 틀 (방문·저장·리뷰 수 기록표 + 쉬운 해석 칸)
```

## 부록: 구글 비즈니스 프로필 (관광·외국인 상권 한정)

상권에 외국인 유동이 있으면 이 부록을 함께 실행한다. 내국인 로컬 검색은 네이버가 지배하므로 그 외 상권에서는 우선순위 낮음.

근거: 2024년 방한 외래관광객 1,637만 명(+48.4%, 이데일리 2025-02), 2025년 1,870만 돌파 (스마트투데이); "외국인 관광객은 도착하자마자 구글맵을 켠다" (로컬루어 2025-10, blog.locallure.kr/google-map-tourist-guide).

- 등록: business.google.com/add → 비즈니스 이름/카테고리/위치/연락처 → 인증. 무료 (공식: support.google.com/business/answer/2911778)
- 이미 지도에 있으면 "비즈니스에 대한 소유권 주장" → 인증 (소유권 미확인 방치 = 제3자 수정에 무방비)
- 산출물: 영문 상호·카테고리(세분화: "Korean BBQ Restaurant" 수준)·영업시간 + 영문 소개 750자 + 영/일/중 메뉴 번역표(메뉴당 1줄) + 외국인 리뷰 영어 답글 템플릿 6종 + 사진 10장 업로드 순서
- 한글로만 등록하면 영문 검색에서 탈락 — 초보 실수 1위 (로컬루어, 위 출처)
