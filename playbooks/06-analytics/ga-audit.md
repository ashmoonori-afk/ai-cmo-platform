# ga-audit

## 목적

GA4 설정을 속성 / 스트림 / 이벤트 / 전환 / 잠재고객 기준으로 진단하여 100점 스코어카드와 우선순위 개선 로드맵을 생성한다.
참조: `ga-optimizer/CLAUDE.md` — GA4 전문 엔지니어 관점의 Measurement Plan First 원칙 적용.

## 에이전트 조합

```
data-analyst
```

단일 에이전트. 사용자가 제공한 GA4 설정 스크린샷 또는 export 데이터 기반 분석.

## 입력

```
BRAND: {brand}               # 브랜드명
GA4_PROPERTY_ID: {id}        # GA4 속성 ID (예: 123456789)
GTM_CONTAINER_ID: {id}       # GTM 컨테이너 ID (선택, 예: GTM-XXXXXXX)
SITE_URL: {url}              # 사이트 URL
BUSINESS_GOALS: {goals}      # 비즈니스 목표 (예: 구매완료,견적신청,회원가입)
SCREENSHOTS: {paths}         # GA4 Admin 스크린샷 경로 목록 (선택)
```

### 민감정보 입력 Gate

- GA4/GTM 스크린샷과 export는 계정 이메일, 사용자 ID, client ID, 쿠키, 토큰, API 키, 인증 헤더를 제거하거나 마스킹한 사본만 사용한다.
- 사용자 단위 이벤트 행이나 잠재고객 raw row는 산출물에 붙이지 않는다.
- 결과에는 설정 상태, 집계 점수, 안전한 로컬 경로만 남긴다.
- 민감정보가 제거되지 않은 입력이면 중단하고 redacted working copy를 요청한다.

## 참조 문서

- `ga-optimizer/CLAUDE.md` — GA4 전문 엔지니어 행동 원칙 및 스킬 아키텍처
- `clients/{brand}/config.md` — 비즈니스 목표, KPI, ICP
- `knowledge-base/{brand}/insights.md` — 이전 GA4 감사 기록

## 프레임워크

### Measurement Plan 수립 (선행 작업)

```
감사 전 비즈니스 목표 정렬:
1. 비즈니스 목표 확인 (config.md 또는 사용자 입력)
2. KPI 정의 (목표별 측정 지표)
3. 측정 이벤트 매핑 (KPI → GA4 이벤트명)
4. 현재 GA4 설정과 Measurement Plan 갭 분석
```

### 영역 1: 속성 기본 설정 (Property Setup) — 20점

```
1-1. 속성 설정 (10점)
  [ ] 속성 이름 명확 (브랜드명 포함) (1점)
  [ ] 시간대 정확 설정 (Asia/Seoul) (2점)
  [ ] 통화 KRW 설정 (2점)
  [ ] 산업 카테고리 적절히 설정 (1점)
  [ ] 데이터 보존 기간 14개월 설정 (2점)
  [ ] Google Signals 활성화 (1점)
  [ ] 사용자 ID 설정 (로그인 서비스인 경우) (1점)

1-2. 연결 설정 (10점)
  [ ] Google Ads 연결 (해당 시) (3점)
  [ ] Search Console 연결 (3점)
  [ ] BigQuery Export 설정 (2점)
  [ ] Looker Studio 연결 (2점)
```

### 영역 2: 데이터 스트림 (Data Streams) — 20점

```
2-1. 웹 스트림 (15점)
  [ ] 올바른 측정 ID (G-XXXXXXXX) 설치 확인 (3점)
  [ ] 향상된 측정 활성화 (스크롤/외부링크/동영상) (4점)
  [ ] 교차 도메인 측정 설정 (해당 시) (2점)
  [ ] 내부 트래픽 필터 설정 (IP 제외) (3점)
  [ ] 개발자/테스트 트래픽 제외 (3점)

2-2. 앱 스트림 (5점, 해당 시)
  [ ] iOS/Android 스트림 설정 (3점)
  [ ] Firebase 연결 (2점)
  (앱 없는 경우 5점 만점 자동 부여)
```

### 영역 3: 이벤트 설정 (Events) — 25점

```
3-1. 자동 수집 이벤트 검증 (8점)
  [ ] page_view 정상 수집 (2점)
  [ ] session_start 정상 수집 (2점)
  [ ] first_visit 정상 수집 (2점)
  [ ] user_engagement 정상 수집 (2점)

3-2. 추천 이벤트 (GA4 Recommended Events) (12점)
  비즈니스 목표별 필수 이벤트:
  [ ] purchase (전자상거래) (4점)
  [ ] add_to_cart (4점)
  [ ] generate_lead (리드 발생) (4점)
  [선택] view_item, begin_checkout, login 등

3-3. 커스텀 이벤트 품질 (5점)
  [ ] 이벤트명 snake_case 규칙 준수 (2점)
  [ ] 파라미터 구조 일관성 (2점)
  [ ] 불필요한 중복 이벤트 없음 (1점)
```

### 영역 4: 전환 설정 (Conversions) — 20점

```
4-1. 전환 이벤트 설정 (15점)
  [ ] 핵심 비즈니스 목표 이벤트 → 전환 표시 (5점)
  [ ] 마이크로 전환 설정 (중간 단계) (3점)
  [ ] 전환 중복 계산 방지 (count_once 설정) (3점)
  [ ] 전환 가치(value) 파라미터 설정 (4점)

4-2. 목표 기간 설정 (5점)
  [ ] 전환 집계 기간 적절 (기본 30일 vs 비즈니스 사이클) (3점)
  [ ] Google Ads 전환 가져오기 설정 (2점)
```

### 영역 5: 잠재고객 & 리포팅 (Audiences & Reporting) — 15점

```
5-1. 잠재고객 설정 (8점)
  [ ] 핵심 세그먼트 정의 (구매자/비구매자/이탈자 등) (4점)
  [ ] 리마케팅 목록 설정 (Google Ads 연동) (2점)
  [ ] 예측 잠재고객 활성화 (해당 시) (2점)

5-2. 탐색 분석 / 리포팅 (7점)
  [ ] 사용자 정의 리포트 설정 (3점)
  [ ] Looker Studio 대시보드 연결 (2점)
  [ ] 채널 그룹 커스텀 설정 (2점)
```

### 점수 등급

```
90-100점: 최적화 완료 — GA4 전문 설정
75-89점:  양호 — 보완 항목 소수
60-74점:  보통 — 중요 설정 누락
40-59점:  미흡 — 핵심 설정 오류 다수
0-39점:   위험 — 데이터 신뢰성 낮음
```

### 개선 우선순위 결정

```
Priority 1 — 데이터 신뢰성 (즉시):
  - 잘못된 측정 ID, 중복 설치, 필터 미설정
  → 데이터 오염 → 즉시 수정

Priority 2 — 전환 추적 완결 (1주):
  - 핵심 비즈니스 전환 미측정
  → ROI 계산 불가 → 1주 내 설정

Priority 3 — 최적화 (1개월):
  - 추천 이벤트 추가, 잠재고객 정교화
  → 분석 품질 향상
```
