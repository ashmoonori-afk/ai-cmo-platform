# A/B 카피 테스트 플레이북

## 목적

동일한 핵심 메시지를 2–3가지 톤·각도·구조로 변형하여 어떤 버전이 타겟 고객에게 가장 강하게 반응하는지 테스트할 수 있는 카피 세트를 제작한다. 감·경험 의존에서 데이터 기반 카피 최적화로 전환한다.

적용 범위: 랜딩페이지 헤드라인, 소셜 포스트, 이메일 제목, 광고 카피, CTA 버튼

---

## 에이전트 조합

```
copywriter (2–3개 병렬) → reviewer
```

| 순서 | 에이전트 | 역할 | 모델 |
|------|---------|------|------|
| 1a | copywriter #1 | 버전 A 작성 (전문적/권위 톤) | sonnet |
| 1b | copywriter #2 | 버전 B 작성 (친근/공감 톤) | sonnet |
| 1c | copywriter #3 | 버전 C 작성 (긴급/이익 강조 톤) | sonnet |
| 2 | reviewer | 3개 버전 검토 + 추천 버전 선정 | sonnet |

> copywriter #1, #2, #3은 동시에 병렬 실행한다.

---

## 입력

| 항목 | 설명 | 필수 여부 |
|------|------|---------|
| `client` | 클라이언트 폴더명 | 필수 |
| `core_message` | 테스트할 핵심 메시지 1개 | 필수 |
| `format` | 카피 형식 (headline / social / email-subject / ad / cta) | 필수 |
| `icp` | 타겟 독자 설명 (없으면 config.md ICP 사용) | 선택 |
| `versions` | 생성할 버전 수 (2 또는 3, 기본: 3) | 선택 |
| `tones` | 테스트할 톤 지정 (없으면 표준 3가지 사용) | 선택 |
| `funnel_stage` | 퍼널 단계 (awareness / consideration / conversion) | 선택 |
| `character_limit` | 글자 수 제한 (형식에 따라 자동 설정) | 선택 |

## 프레임워크

### 표준 3가지 톤 변형

```
버전 A — 전문적 / 권위 톤:
- 데이터·수치·전문 용어 활용
- 이성적 판단에 소구
- 신뢰·전문성·안정성 강조
- 예: "87%의 마케터가 선택한 방법"

버전 B — 친근 / 공감 톤:
- 독자의 감정·상황에 공감
- 쉬운 언어, 대화체
- 관계·커뮤니티·소속감 강조
- 예: "저도 같은 고민 했었어요. 이렇게 해결했습니다"

버전 C — 긴급 / 이익 강조 톤:
- 즉각적 가치 또는 손실 회피 강조
- FOMO(기회 손실 두려움) 활용
- 명확한 혜택·절약·ROI 중심
- 예: "지금 신청하면 30% 절약 + 무료 컨설팅"
```

### 형식별 글자 수 기준

```
headline (랜딩페이지 헤드라인): 20–40자
social (소셜 포스트 첫 줄): 40–80자
email-subject (이메일 제목): 20–40자
ad (광고 카피): 15–30자
cta (CTA 버튼): 5–15자
```

### 각도(Angle) 변형 옵션

```
톤 외에 각도(강조 포인트)도 변형 가능:
- 각도 1 — 문제 중심: "이 문제 겪고 있나요?"
- 각도 2 — 솔루션 중심: "이렇게 하면 됩니다"
- 각도 3 — 결과 중심: "이런 성과를 냈습니다"
- 각도 4 — 호기심 중심: "아무도 말 안 해준 사실"
- 각도 5 — 사회적 증거 중심: "000명이 선택했습니다"
```

### reviewer: 추천 기준

```
reviewer는 다음 기준으로 각 버전을 평가하고 추천 버전을 선정한다:

1. ICP 적합성: 타겟 독자의 심리·상황에 얼마나 맞는가 (5점)
2. 메시지 명확성: 핵심 메시지가 즉각적으로 전달되는가 (5점)
3. 행동 유발력: 독자가 다음 행동을 하고 싶어지는가 (5점)
4. 브랜드 일관성: 브랜드 가이드라인을 준수하는가 (5점)
5. 차별성: 경쟁사 대비 독특한 포인트가 있는가 (5점)

총점 + 추천 버전 + 추천 이유 (2–3줄)
```
