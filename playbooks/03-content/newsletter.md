# 뉴스레터 플레이북

## 목적

구독자와의 정기적인 접점을 만들어 브랜드 신뢰도를 쌓고, 유용한 정보 제공으로 열람률(Open Rate)과 클릭률(CTR)을 높이며, 구독자를 잠재 고객으로 전환한다.

목표 지표: 열람률 30%+, 클릭률 5%+

---

## 에이전트 조합

```
copywriter → reviewer
```

| 순서 | 에이전트 | 역할 | 모델 |
|------|---------|------|------|
| 1 | copywriter | 뉴스레터 전체 원고 작성 (제목·인트로·섹션 3개·CTA) | sonnet |
| 2 | reviewer | 구조·브랜드·가독성 검증 | sonnet |

---

## 입력

| 항목 | 설명 | 필수 여부 |
|------|------|---------|
| `client` | 클라이언트 폴더명 | 필수 |
| `edition` | 호수 또는 날짜 (예: 2026-03 / Vol.12) | 필수 |
| `main_topic` | 이번 호 메인 주제 | 필수 |
| `sections` | 섹션별 주제 3개 (없으면 copywriter가 선정) | 선택 |
| `cta_destination` | CTA 링크 목적지 | 선택 |
| `highlight` | 특별히 강조할 소식·프로모션 | 선택 |

---

| 문서 | 경로 | 목적 |
|------|------|------|

## 프레임워크

### 뉴스레터 구조

```
[제목 라인] — 호기심 유발, 28자 이내 (모바일 미리보기 최적화)
[프리헤더] — 제목 보완, 50자 이내

[인트로] — 이번 호 요약 + 독자와의 친근한 연결, 100자 이내

[섹션 1] — 메인 주제 (300자, 독립적으로 읽혀도 가치 있는 내용)
[섹션 2] — 보조 주제 (300자)
[섹션 3] — 업계 소식 또는 팁 (300자)

[CTA] — 단일 CTA, 명확한 행동 유도
[푸터] — 수신거부 링크, 회사 정보
```

### 제목 라인 작성 원칙
```
- 숫자 포함: "3가지 방법으로 전환율 2배 올린 사례"
- 호기심 유발: "아무도 말 안 해준 마케팅 자동화의 함정"
- 독자 이익 중심: "이번 주 놓치면 안 될 업계 트렌드 5가지"
- A/B 테스트용 제목 2개 제공 (메인 + 대안)
```

### 섹션 작성 원칙
```
각 섹션은 독립적으로 읽혀도 가치 있어야 함
- 섹션 제목: 5–10자 이내 소제목
- 본문: 300자 내외
- 섹션 말미: "더 알아보기" 링크 또는 연관 CTA (선택)

섹션 유형 예시:
- [인사이트] 업계 트렌드 분석
- [HOW-TO] 실행 가능한 팁
- [사례] 고객 또는 업계 성공 사례
- [도구] 추천 툴 또는 리소스
- [소식] 자사 제품 업데이트·프로모션
```

### CTA 작성 원칙
```
- 1개의 주요 CTA만 사용 (이메일당 CTA 분산 금지)
- 버튼 텍스트: 동사로 시작, 10자 이내
- CTA 앞 문장: CTA를 클릭해야 하는 이유 1줄
```

---

## {섹션 2 제목}

{본문 300자}

{선택: [더 알아보기]({링크})}

---

## {섹션 3 제목}

{본문 300자}

{선택: [더 알아보기]({링크})}
