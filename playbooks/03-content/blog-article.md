# 블로그 아티클 플레이북

## 목적

SEO 최적화된 블로그 아티클을 작성하여 검색 유입 트래픽을 확보하고, 타겟 독자(ICP)에게 가치 있는 정보를 제공함으로써 브랜드 신뢰도와 리드 전환율을 높인다.

목표 글자 수: **1500–3000자**
우선 성과 지표: 유기 검색 순위, 페이지 체류 시간, CTA 클릭률

---

## 에이전트 조합

```
seo-specialist → copywriter → reviewer
```

| 순서 | 에이전트 | 역할 | 모델 |
|------|---------|------|------|
| 1 | seo-specialist | 타겟 키워드 선정, 경쟁 페이지 분석, SEO 브리프 작성 | sonnet |
| 2 | copywriter | SEO 브리프 기반 블로그 원고 작성 | sonnet |
| 3 | reviewer | 구조·브랜드·SEO 체크리스트 검증 | sonnet |

---

## 입력

| 항목 | 설명 | 필수 여부 |
|------|------|---------|
| `topic` | 블로그 주제 또는 목표 키워드 | 필수 |
| `client` | 클라이언트 폴더명 | 필수 |
| `target_keyword` | 메인 타겟 키워드 (없으면 seo-specialist가 선정) | 선택 |
| `cta_destination` | CTA 링크 목적지 (랜딩페이지, 상담 신청 등) | 선택 |
| `reference_url` | 참고할 경쟁사 글 URL | 선택 |
| `tone` | 톤 지정 (없으면 brand-guidelines.md 따름) | 선택 |

## 프레임워크

### Step 1 — seo-specialist: SEO 브리프 작성

```
1. 타겟 키워드 1개 선정 (검색량·난이도·전환 잠재력 기준)
2. 보조 키워드 3–5개 선정
3. 경쟁 상위 3개 페이지 구조 분석
4. 추천 제목(H1) 후보 3개 작성
5. H2 구조 초안 작성 (최소 4개)
6. 내부 링크 연결 가능한 기존 페이지 2개 이상 제안
```

### Step 2 — copywriter: 블로그 원고 작성

```
블로그 구조 (1500–3000자):

1. Hook + 직답 (100–200자)
   - 독자가 공감하는 문제 상황 또는 강렬한 질문으로 시작
   - 타겟 키워드를 자연스럽게 포함
   - **첫 H2 이전에 질문에 대한 직답(정의·결론) 문단 필수** — AI 검색이
     발췌하는 단위 (GEO — prompts/shared/geo-checklist.md)

2. 문제 정의 (300–500자)
   - 왜 이 문제가 중요한지, 방치하면 어떤 결과가 생기는지
   - 독자의 현실 상황과 연결

3. 해결책 (500–800자)
   - 단계별 방법론 또는 핵심 원칙 3–5가지
   - H2 소제목 사용, 실행 가능한 내용 중심

4. 증거 / 사례 (300–500자)
   - 수치 데이터, 고객 사례, 업계 리서치 결과
   - **수치 = 출처 URL 필수** (없으면 [추정] 태그 또는 삭제)
   - 통계 문장 2개+ 및 인용문 1개+ 포함 (GEO: 통계 +31%, 인용문 +41% — 논문 근거)

5. 실행 팁 (200–300자)
   - 독자가 오늘 당장 적용할 수 있는 액션 2–3가지

6. CTA (50–100자)
   - 다음 행동 유도 (상담 신청 / 무료 체험 / 가이드 다운로드 등)
   - 링크 또는 버튼 텍스트 포함
```

### Step 3 — reviewer: SEO 체크리스트 검증

```
SEO 체크리스트:
- [ ] 타겟 키워드가 제목(H1), 첫 문단, H2 1개 이상에 포함
- [ ] 메타 설명 (150자 내외) 작성 완료
- [ ] H2 최소 4개 사용
- [ ] 내부 링크 제안 2개 이상 포함
- [ ] 이미지 alt 텍스트 제안 포함 (이미지별)
- [ ] 글자 수 1500–3000자 범위 내
- [ ] 브랜드 톤앤매너 준수
- [ ] placeholder / TO[D]O 없음

GEO 체크 (prompts/shared/geo-checklist.md):
- [ ] 첫 H2 이전 직답 문단 (40–60단어)
- [ ] 출처 있는 통계 문장 2개+ / 인용문 1개+ / 정의문 1개+
- [ ] 키워드 스터핑 없음 (GEO 가시성 −10% + 구글 스팸 정책)
```

### 리프레시 규칙 (발행 후 운영)

- 발행 6개월 후부터 **트래픽 상위 20% 글을 분기별 갱신** (수치 최신화, 새 사례 추가, 갱신일 표기)
- 근거: 기존 글 갱신으로 오가닉 +106%·리드 2배 (HubSpot); 콘텐츠 부패 6개월마다 클릭 20-30% 감소 [2차]; AI 검색은 더 신선한 콘텐츠 인용 경향 [2차]

## 출력 템플릿

```markdown
---
타겟 키워드: {keyword}
메타 설명: {150자 이내 메타 설명}
추천 내부 링크:
  1. {링크 텍스트} → {URL 또는 페이지 제목}
  2. {링크 텍스트} → {URL 또는 페이지 제목}
---

# {H1 제목 — 타겟 키워드 포함}

{Hook: 100–200자, 독자 문제 공감}

---

## {H2: 문제 정의}

{문제 설명 300–500자}

## {H2: 해결책 또는 방법론}

{해결책 설명 500–800자}

### {H3 세부 단계 1}
{내용}

### {H3 세부 단계 2}
{내용}

## {H2: 실제 사례 / 데이터}

{증거·사례 300–500자}
> 출처: {출처명, 연도} 또는 [추정]

## {H2: 바로 적용하는 방법}

{실행 팁 200–300자}

---

**{CTA 텍스트 — 50–100자}**
[{버튼 텍스트}]({CTA URL})
```
