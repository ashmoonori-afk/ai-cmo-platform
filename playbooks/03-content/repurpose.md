# 콘텐츠 리퍼포징 플레이북

## 목적

원본 콘텐츠 1개(블로그 아티클, 케이스스터디, 영상 스크립트 등)에서 핵심 메시지를 추출하여 LinkedIn, Twitter, 뉴스레터, 카드뉴스 4개 포맷으로 병렬 변환함으로써, 최소 제작 비용으로 최대 채널 커버리지를 달성한다.

---

## 에이전트 조합

```
repurposer (4개 병렬) → reviewer
```

| 순서 | 에이전트 | 역할 | 모델 |
|------|---------|------|------|
| 1a | repurposer #1 | 원본 → LinkedIn 포스트 변환 | haiku |
| 1b | repurposer #2 | 원본 → Twitter 스레드 변환 | haiku |
| 1c | repurposer #3 | 원본 → 뉴스레터 섹션 변환 | haiku |
| 1d | repurposer #4 | 원본 → 카드뉴스 슬라이드 카피 변환 | haiku |
| 2 | reviewer | 4개 포맷 일관성·브랜드·플랫폼 규격 검증 | sonnet |

> repurposer #1–4는 동시에 병렬 실행한다.

---

## 입력

| 항목 | 설명 | 필수 여부 |
|------|------|---------|
| `client` | 클라이언트 폴더명 | 필수 |
| `source_content` | 원본 콘텐츠 전문 또는 파일 경로 | 필수 |
| `source_type` | 원본 유형 (blog / case-study / script / report) | 필수 |
| `core_message` | 핵심 메시지 (없으면 repurposer가 추출) | 선택 |
| `target_platforms` | 변환 대상 플랫폼 (기본: all 4개) | 선택 |
| `cta_url` | 원본 콘텐츠 링크 (각 포맷 CTA에 사용) | 선택 |

---

## 참조 문서

| 문서 | 경로 | 목적 |
|------|------|------|
| 클라이언트 설정 | `clients/{client}/config.md` | 업종·ICP 파악 |

## 프레임워크

### Step 0 — 핵심 메시지 추출 (repurposer 공통 사전 작업)

```
원본 콘텐츠에서 추출:
1. 핵심 메시지 3개 (각 1문장)
   - 메시지 1: 주요 인사이트 또는 주장
   - 메시지 2: 핵심 근거 또는 데이터
   - 메시지 3: 독자 액션 또는 결론

2. 재사용 가능한 수치·데이터 목록
3. 인용 가능한 강렬한 문장 1–3개
```

### Step 1a — LinkedIn 포스트 변환

```
규격: 300–500자
구조: Hook(1–2줄) → 핵심 내용(3–6줄) → CTA(1줄) → 해시태그(3–5개)

변환 원칙:
- 원본의 가장 강렬한 인사이트를 Hook으로
- 블로그 구조를 4–6개의 짧은 단락으로 압축
- "전문 글" 읽는 느낌 유지
- 원본 링크 CTA: "전문은 댓글 링크에서"
```

### Step 1b — Twitter 스레드 변환

```
규격: 5–7트윗, 각 280자 이내
구조:
  트윗 1: Hook — 결론 또는 가장 놀라운 포인트 먼저
  트윗 2–5: 근거, 단계, 사례 (핵심 메시지 분배)
  트윗 6: 요약 또는 실행 팁
  트윗 7: CTA + 원본 링크

변환 원칙:
- 각 트윗이 독립적으로 읽혀도 의미 있어야 함
- 트윗 번호 표기 (1/ 2/ ...)
- 해시태그 트윗 7에만 1–2개
```

### Step 1c — 뉴스레터 섹션 변환

```
규격: 섹션 1개, 300자 내외
구조: 소제목 → 본문(250자) → "더 알아보기" 링크

변환 원칙:
- 원본의 가장 실용적인 부분 선택
- 뉴스레터 독자 수준에 맞게 쉽게 재작성
- 마지막에 원본 링크 유도
```

### Step 1d — 카드뉴스 슬라이드 카피 변환

```
규격: 5–8장 슬라이드, 각 슬라이드 50자 이내 제목 + 100자 이내 본문
구조:
  슬라이드 1 (표지): 제목 + 부제
  슬라이드 2–6: 핵심 내용 각 1개씩
  슬라이드 7 (마지막): 요약 + CTA

변환 원칙:
- 각 슬라이드는 하나의 아이디어만
- 짧고 임팩트 있게 (긴 문장 금지)
- 시각적 요소 방향 제안 포함 (예: [그래프 이미지])
```
