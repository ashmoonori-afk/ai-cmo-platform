# 후킹문안 리스트 플레이북

## 목적

클라이언트의 사업 정보와 전략 산출물을 바탕으로, **어디에나 바로 붙여 쓸 수 있는 후킹문안**(고객의 눈길을 붙잡는 한 줄 카피) 리스트를 한 번에 만든다. 랜딩 히어로, SNS, 광고, 전단에 흩어져 있던 카피를 **채널별로 정리한 한 장짜리 문안집**이 최종 산출물이다.

적용 범위: 홈페이지 히어로/헤드라인, 네이버·구글 검색 광고, SNS 게시물 첫 줄, 오프라인(전단/현수막/매장 안내), 문자·카카오톡 메시지

---

## 에이전트 조합

```
copywriter → reviewer
```

| 순서 | 에이전트 | 역할 | 모델 |
|------|---------|------|------|
| 1 | copywriter | 채널별 후킹문안 작성 | sonnet |
| 2 | reviewer | 과장·허위·브랜드 위반 검증 + Top 5 선정 확인 | sonnet |

---

## 입력

| 항목 | 설명 | 필수 여부 |
|------|------|---------|
| `client` | 클라이언트 폴더명 | 필수 |
| `strategy_ref` | 전략 문서 경로 (예: `outputs/{client}/strategy/...`) — 있으면 핵심 메시지를 여기서 가져온다 | 선택 |
| `channels` | 문안을 만들 채널 목록 (없으면 config.md의 채널 + 아래 표준 5채널) | 선택 |
| `count_per_channel` | 채널당 문안 수 (기본 3) | 선택 |

표준 5채널: 홈페이지 히어로 / 검색(네이버·구글) / SNS / 오프라인(전단·현수막) / 메시지(문자·카카오톡)

## 프레임워크

### Step 1: 소재 추출

config.md에서 problem / 차별점 / proof / CTA를 뽑는다. `strategy_ref`가 있으면 전략 문서의 핵심 메시지를 병합한다. **증거(proof)가 없는 수치는 소재로 쓰지 않는다.**

### Step 2: 후킹 공식 적용

채널당 `count_per_channel`개를 만들되, 한 채널 안에서 서로 다른 공식을 쓴다. 톤·각도의 세부 변형은 `ab-copy.md`의 표준 3톤·5각도를 따른다.

후킹 공식 8종:

```
1. 문제 직격:   "{고객의 불편}, 아직도 참고 계세요?"
2. 숫자 증거:   "{검증된 수치} — {증거 한 줄}"        ← proof 있는 경우만
3. 손실 회피:   "모르면 {잃는 것}"
4. 비교 우위:   "{기존 대안}보다 {차별점}"
5. 호기심:      "{타겟}이 {행동} 전에 꼭 확인하는 것"
6. 사회적 증거: "{N명·N곳}이 이미 {행동}"             ← proof 있는 경우만
7. 즉시 이익:   "오늘 {행동}하면 {혜택}"
8. 공감 스토리: "저도 {같은 고민}에서 시작했습니다"
```

### Step 3: 채널별 형식 맞춤

글자 수 기준 (`ab-copy.md` 기준 재사용 + 오프라인/메시지 추가):

```
홈페이지 히어로: 20–40자
검색 광고:       15–30자
SNS 첫 줄:       40–80자
오프라인:        15–25자 (멀리서 3초 안에 읽히는 길이)
문자·카카오톡:   30–60자 (첫 화면 미리보기에 잘리지 않는 길이)
```

### Step 4: 사용법 한 줄

각 문안에 다음 두 가지를 **쉬운 말로** 붙인다. 마케팅 용어 금지.

- **어디에**: "가게 앞 입간판에", "네이버 광고 제목에", "카톡 단골 알림에"
- **왜 먹히는지**: "손님이 자기 얘기라고 느껴서", "숫자가 눈에 먼저 들어와서"

### Step 5: reviewer 검증

- `prompts/shared/gate-check.md` 기본 체크
- 브랜드 금지 표현 위반 여부
- **과장·허위 청소**: 증거 없는 수치·최상급("최고", "1위")은 삭제하거나 [추정] 태그
- Top 5 선정이 서로 다른 채널·공식을 포함하는지 확인
