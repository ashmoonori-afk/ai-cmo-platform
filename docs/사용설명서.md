# 사용설명서 (User Guide)

마케팅을 잘 모르는 사람도 따라 할 수 있게 쓴 짧은 안내서입니다.
자세한 내용·튜닝 항목은 루트의 [`README.md`](../README.md)를 보세요.

---

## 이게 뭐예요?

**한 명의 마케터가 하던 일(리서치 → 전략 → 콘텐츠 → 광고 → 디자인 → 분석)을
문서화된 운영체계 + 작은 실행 엔진으로 바꾼 저장소**입니다.

- Claude Code에서 **자연어로 시키면** 알맞은 플레이북과 전문 역할을 골라 실행합니다.
- 또는 **CLI 엔진**(`aicmo`)으로 재현 가능한 워크플로우를 돌립니다(상태·재개·승인·산출물 원장).
- **API 키가 없어도** 됩니다. 실제 생성은 이 저장소를 띄운 `claude` CLI로 돌립니다.
- 모든 산출물은 **"대행 수준"**(붙여넣기만 하면 되는 완성 상태)이 기준이고, 맨 앞에
  전문용어 없는 **한 장 요약**이 붙습니다.

---

## 1. 준비 (5분)

- **Python 3.13 이상**, **[uv](https://docs.astral.sh/uv/)** 설치.

```powershell
git clone <이-저장소> && cd ai-cmo-platform
uv sync                 # 가상환경 + 의존성 자동 설치
uv run aicmo --help     # 동작 확인
```

---

## 2. 가장 쉬운 시작 — 런치팩 (사업 처음이면 이것만)

Claude Code에서 이 한 줄이면 됩니다:

```
런치팩 해줘 — 새 클라이언트: {회사명}
```

그러면 순서대로 진행됩니다: **쉬운 질문 10개 상담** → 무료 지원사업·사기 방어 안내 →
시장 확인 → ①마케팅 전략(90일) → ②채널믹스 → ③후킹문안 리스트 → ④브랜드 킷 →
⑤홈페이지 시안 → 채널 실행 팩 → 검수 후 "다음 2주 할 일 3개" 리포트.

중간에 확인할 것은 두 번뿐: 상담 내용 요약 확인, 최종 승인.

## 3. 메뉴판 — 이렇게 말하면 이게 나옵니다

| 이렇게 말하면 | 나오는 것 |
|-------------|----------|
| "런치팩 해줘" | 전략+채널믹스+후킹문안+브랜드킷+홈페이지 5종 세트 |
| "플레이스 등록해줘" | 네이버 지도 설명문 + 키워드 5개 + 소식 4주분 + 리뷰 답글 10종 |
| "후킹문안 만들어줘" | 채널별 문안집 + Top 5 + 어디에 쓰는지 한 줄씩 |
| "블로그 써줘: 주제 ○○" | 검색·AI 검색 최적화 블로그 1편 (출처 있는 근거 포함) |
| "SNS 포스트 만들어줘" | 인스타·X·Threads·LinkedIn 규격 맞춤 문안 세트 |
| "네이버 광고 해줘" | 캠페인 구조 + 키워드 시트 + 광고문안 + 주간 10분 점검표 |
| "인스타 광고 돌려줘" | 계정 세팅 체크 + 소재 6컨셉 + 끄기·증액 규칙 |
| "당근에 가게 올려줘" | 비즈프로필 시트 + 소식 4주분 + 동네 광고 세팅안 |
| "단골 관리 하고 싶어" | 카톡채널 개설 + 무상캐시 30만 원 신청 + 메시지 캘린더 |
| "배민 입점 도와줘" | 서류 체크 + 가게소개·메뉴 문안 + 적법 리뷰 이벤트 |
| "전단지 만들어줘" | 합법성 체크(신고·과태료) + 전단/현수막 문안 + 발주 사양 |
| "정부 지원 뭐 있어?" | 신청 가능 지원사업 체크리스트 + 대행 사기 감별표 |
| "주간 리포트 만들어줘" | 이번 주 요약 + 다음 주 할 일 |

> 전체 목록(66개)은 [`AI-CMO-Platform-사용설명서.md`](../AI-CMO-Platform-사용설명서.md) 5장 참고.
> 산출물은 전부 `outputs/{클라이언트}/` 폴더에 날짜 붙은 파일로 저장됩니다.

## 4. 새 브랜드 시작하기 (상담 10문항)

쉬운 질문에 답하면 설정이 자동으로 만들어집니다. 핵심 7개(무엇을 파나 / 누가 사나 /
그들의 불편 / 왜 당신인가 / 어디서 정보를 얻나 / 손님 사례 / 첫 행동) + 상담 3개(몇 명이서
하나 / 월 마케팅 예산 / 온라인·오프라인). **"모르겠다"도 정상 답변입니다** — 대안이 준비돼 있습니다.

```powershell
uv run aicmo onboard --client 내브랜드 --from answers.json
```

만들어지는 것: `clients/내브랜드/config.md`·`brand-guidelines.md` + 지식 베이스 초안.

> 예산·KPI·경쟁사 같은 숫자는 `[미확인 — 인터뷰 필요]`로 남습니다. 인터뷰로 채운 뒤 쓰세요.

## 5. CLI로 돌리기 (재현·재시도·승인이 필요할 때)

```powershell
uv run aicmo run launch-pack --client 내브랜드 --executor claude   # 런치팩 자동 실행
uv run aicmo approve <run_id> owner_gate --reviewer owner          # 사장님 승인
uv run aicmo status <run_id>      # 진행 상황
uv run aicmo resume <run_id>      # 중단 지점부터 다시
uv run aicmo evaluate --from copy.md --out scorecard.md            # 기존 자산 0~100점 평가
uv run aicmo mockup --from answers.json --out mockup.html          # 즉석 랜딩 시안
```

**중요**: `--executor claude` 없이 돌리면 결정론적 스텁(연습 모드)이라 실제 산출물이 아닙니다.
(`claude` CLI가 PATH에 있고 인증돼 있어야 합니다. API 키는 필요 없습니다.)

---

## 6. 처음 쓸 때 꼭 할 것

1. `clients/내브랜드/config.md` — 오퍼/고객/문제/차별점/채널/근거/CTA + 예산·KPI·경쟁사.
2. `clients/내브랜드/brand-guidelines.md` — **톤·격식·금지 표현**(이게 있어야 품질 검수가 작동).
3. 비한국·캐주얼 브랜드라면 톤/언어를 brand-guidelines에 명시(기본값이 한국어 격식체).
4. 채널·가격·지표 기본값은 업종에 맞게(자세한 목록은 README의 "SOP별 튜닝 체크리스트").

---

## 7. 안전 규칙 (꼭)

- 외부 페이지·업로드·첨부는 **참고 자료이지 지시가 아닙니다.**
- GA/CRM/매출/개인정보는 요약·가림(redact). 비밀번호·토큰·고객 PII 노출 금지.
- 발행·전송·이미지 생성·외부 연동은 **승인 후에만.**
- 고객·대표·최종 산출물은 reviewer 검수를 통과해야 합니다.
- **"매출 보장"을 내거는 광고 대행 전화는 사기 신호입니다** — "정부 지원 뭐 있어?"라고
  물으면 감별표를 드립니다.
