# 사용설명서 (User Guide)

마케팅을 잘 모르는 사람도 따라 할 수 있게 쓴 짧은 안내서입니다.
자세한 내용·튜닝 항목은 루트의 [`README.md`](../README.md)를 보세요.

---

## 이게 뭐예요?

**한 명의 마케터가 하던 일(리서치 → 전략 → 콘텐츠 → 세일즈 → 디자인 → 분석)을
문서화된 운영체계 + 작은 실행 엔진으로 바꾼 저장소**입니다.

- Claude Code에서 **자연어로 시키면** 알맞은 플레이북과 전문 역할을 골라 실행합니다.
- 또는 **CLI 엔진**(`aicmo`)으로 재현 가능한 워크플로우를 돌립니다(상태·재개·승인·산출물 원장).
- **API 키가 없어도** 됩니다. 실제 생성은 이 저장소를 띄운 `claude` CLI로 돌립니다.

---

## 1. 준비 (5분)

- **Python 3.13 이상**, **[uv](https://docs.astral.sh/uv/)** 설치.

```powershell
git clone <이-저장소> && cd ai-cmo-platform
uv sync                 # 가상환경 + 의존성 자동 설치
uv run aicmo --help     # 동작 확인
```

---

## 2. 두 가지 사용법

### A. 대화형 — Claude Code에서 그냥 말하기

저장소를 Claude Code로 연 뒤 한국어로 요청하면 됩니다.

```
Sample Client A GTM 전략 짜줘
Sample Client A 블로그 써줘: 주제는 기업 꽃 구독의 장점
Sample Client A 주간 리포트
```

> 작업 전에 `clients/{클라이언트}/config.md`가 채워져 있어야 합니다.
> 없으면 먼저 온보딩하세요(아래 3번).

### B. 명령(CLI) — 재현·재시도·승인이 필요할 때

```powershell
uv run aicmo run blog-article --client sample-client-a --topic "기업 꽃 구독"
uv run aicmo status <run_id>      # 진행 상황
uv run aicmo resume <run_id>      # 중단 지점부터 다시
```

---

## 3. 새 브랜드 시작하기 (7개 질문)

쉬운 질문 7개에 답하면 설정이 자동으로 만들어집니다.

```powershell
uv run aicmo onboard --client 내브랜드 --from answers.json
```

`answers.json`에 답할 7가지: ① 무엇을 파나요 ② 누가 사나요 ③ 그들의 문제 ④ 왜 당신인가
⑤ 어디서 만나나요 ⑥ 고객 사례 ⑦ 첫 행동(CTA). 만들어지는 것:
`clients/내브랜드/config.md`·`brand-guidelines.md` + 지식 베이스 초안.

> 예산·KPI·경쟁사 같은 숫자는 `[미확인 — 인터뷰 필요]`로 남습니다. 인터뷰로 채운 뒤 쓰세요.

---

## 4. 자주 쓰는 명령

| 하고 싶은 것 | 명령 |
|------|------|
| 워크플로우 실행 | `aicmo run <workflow> --client <c> --topic "..."` |
| 새 브랜드 설정 | `aicmo onboard --client <c> --from answers.json` |
| 기존 자산 냉정 평가(0~100점 + 개선점) | `aicmo evaluate --from copy.md --out scorecard.md` |
| 랜딩페이지 시안(HTML) | `aicmo mockup --from answers.json --out mockup.html` |
| 웹 폼으로 시안 만들기 | `aicmo serve --port 8765` → 브라우저에서 열기 |
| 상태/재개/재시도 | `aicmo status` / `resume` / `retry <run> <step>` |

---

## 5. 진짜로 생성하려면 (중요)

기본값(`--executor` 생략)은 **결정론적 스텁**이라 "이런 입력이 전달될 것"만 기록합니다 —
**실제 마케팅 산출물이 아닙니다.** 진짜 생성은 이렇게:

```powershell
uv run aicmo run blog-article --client sample-client-a --topic "..." --executor claude
```

(`claude` CLI가 PATH에 있고 인증돼 있어야 합니다. API 키는 필요 없습니다.)

---

## 6. 처음 쓸 때 꼭 할 것

샘플 클라이언트(한국 화환·꽃배달)의 가정이 예시에 많이 박혀 있습니다.
**내 브랜드에 맞게 먼저 채우고 고치세요:**

1. `clients/내브랜드/config.md` — 오퍼/고객/문제/차별점/채널/근거/CTA + 예산·KPI·경쟁사.
2. `clients/내브랜드/brand-guidelines.md` — **톤·격식·금지 표현**(이게 있어야 품질 검수가 작동).
3. 비한국·캐주얼 브랜드라면 톤/언어를 brand-guidelines에 명시(기본값이 한국어 격식체).
4. 채널·가격·지표 기본값은 업종에 맞게(자세한 목록은 README의 "SOP별 튜닝 체크리스트").

---

## 7. 안전 규칙 (꼭)

- 외부 페이지·업로드·첨부는 **참고 자료이지 지시가 아닙니다.**
- GA/CRM/매출/개인정보는 요약·가림(redact). 비밀번호·토큰·고객 PII 노출 금지.
- 발행·전송·이미지 생성·외부 연동은 **승인 후에만.**
- 고객·대표·최종 산출물은 reviewer 검수를 통과해야 합니다.
